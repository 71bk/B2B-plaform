# 部署與基礎設施

---

## Docker Compose

第二階段加入 Docker Compose：

```yaml
services:
  backend:
    build: .
    ports:
      - "8080:8080"
    depends_on:
      - postgres
      - redis
      - minio

  postgres:
    image: postgres:16
    ports:
      - "5432:5432"

  redis:
    image: redis:7.2
    ports:
      - "6379:6379"

  minio:
    image: minio/minio
    ports:
      - "9000:9000"
      - "9001:9001"
```

## 環境變數

| 變數 | 說明 |
|---|---|
| `SPRING_PROFILES_ACTIVE` | `local` / `dev` / `prod` |
| `DB_HOST` | PostgreSQL host |
| `DB_PORT` | PostgreSQL port |
| `DB_NAME` | Database name |
| `DB_USERNAME` | Database user |
| `DB_PASSWORD` | Database password |
| `JWT_ACCESS_SECRET` | Access token secret |
| `JWT_REFRESH_SECRET` | Refresh token secret |
| `CORS_ALLOWED_ORIGINS` | 前端網域白名單 |
| `REDIS_HOST` | Redis host |
| `MINIO_ENDPOINT` | MinIO endpoint |

Secrets 不應提交到 repo。

## Nginx

Production 可由 Nginx 代理前端與後端：

```text
/              -> Next.js frontend
/api/*         -> Spring Boot backend
/actuator/*    -> restricted
```

---

## CI/CD

GitHub Actions 建議流程：

```text
Push / Pull Request
    ↓
Run unit tests
    ↓
Run security tests
    ↓
Build jar
    ↓
Build Docker image
    ↓
Deploy
```

MVP 階段至少要能在 CI 執行 Maven test。

---

## 備份

PostgreSQL 備份策略：

1. 每日 pg_dump。
2. 保留最近 7 到 14 天。
3. 定期驗證 restore。
4. 文件儲存需與 DB 備份策略分開處理。
