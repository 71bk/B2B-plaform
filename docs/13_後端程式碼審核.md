# 後端程式碼審核

---

## Review 目標

Code review 優先檢查：

1. 是否破壞權限與資料隔離。
2. 是否允許非法狀態轉換。
3. 是否漏寫 Status Log 或 Audit Log。
4. 是否缺少交易控制。
5. 是否缺少必要測試。

---

## Controller

- 是否只處理 HTTP request / response。
- 是否有 request validation。
- 是否沒有直接操作 Entity 狀態。
- 是否有適當的 `@PreAuthorize`。
- 是否回傳 DTO 而不是 Entity。

---

## Service / Domain

- 狀態轉換是否集中管理。
- 是否檢查角色與資源權限。
- 是否使用 `@Transactional`。
- 是否同時寫入 status log 與 audit log。
- 終止狀態是否不可修改。
- optimistic locking 衝突是否有明確錯誤處理。

---

## Repository / DB

- 查詢是否有對應 index。
- 是否避免 N+1 查詢。
- FK 與 unique constraint 是否由 migration 管理。
- 金額是否使用 `BigDecimal` / `NUMERIC(12,2)`。
- 是否沒有在程式中硬刪重要稽核資料。

---

## Security

- 是否不信任前端傳入 userId。
- 是否使用 SecurityContext 取得目前使用者。
- Sales 是否只能看自己的案件。
- 審核 API 是否限制對應角色。
- Cookie / CORS / CSRF 設定是否合理。
- log 是否沒有輸出 token 或密碼。

---

## Tests

- 狀態機是否有 unit test。
- 角色權限是否有 security test。
- 審核流程是否有 integration test。
- 是否測試非法跳關。
- 是否測試終止狀態不可修改。
- 是否測試 concurrent update / version conflict。
