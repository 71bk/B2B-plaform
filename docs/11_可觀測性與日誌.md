# 可觀測性與日誌

---

## Log 策略

每筆 API Request 建議記錄：

| 欄位 | 說明 |
|---|---|
| traceId | request correlation id |
| userId | 使用者 ID |
| organizationId | 目前租戶 ID，平台級 request 可為空 |
| method | HTTP method |
| path | request path |
| status | HTTP status |
| durationMs | 處理時間 |
| errorCode | 錯誤碼 |

不要在 log 中輸出密碼、token、完整 cookie 或敏感文件內容。

---

## TraceId / MDC

每次 request 進入系統時建立 traceId：

1. 若 header 有 `X-Trace-Id`，可沿用。
2. 若沒有，後端產生 UUID。
3. traceId 放入 MDC。
4. API response 也回傳 traceId。
5. Audit Log 寫入 traceId。

---

## Metrics

可由 Spring Boot Actuator 提供：

```text
/actuator/health
/actuator/metrics
/actuator/info
```

關注指標：

| 指標 | 說明 |
|---|---|
| API latency | API 回應時間 |
| Error rate | 錯誤率 |
| Login failure count | 登入失敗次數 |
| Pending case count | 待審核案件數 |
| Average processing time | 平均處理時間 |
| DB query latency | DB 查詢耗時 |

---

## Audit Correlation

`operation_audit_logs.trace_id` 與 `operation_audit_logs.organization_id` 需能對應 request log 的 traceId 與 organizationId，方便追查：

```text
API log traceId
    ↓
operation_audit_logs.trace_id
    ↓
operation_audit_logs.organization_id
    ↓
refund_case_status_logs
```

---

## 告警建議

第二階段可補：

1. 5xx error rate 過高。
2. 登入失敗次數異常。
3. DB connection pool 耗盡。
4. pending cases 超過門檻。
5. 平均處理時間超過 SLA。
